# **서비스 기획서: 영화 선착순 할인 이벤트 통합 대시보드 (가칭: MovieEventCalender)**

## **1\. 서비스 개요**

* **목적:** CGV(스피드쿠폰), 메가박스(빵원티켓), 롯데시네마(무비싸다구) 3사의 선착순 영화 할인 이벤트 정보를 단일 웹페이지에서 제공한다.  
* **핵심 가치:** 사용자가 이벤트 일정을 '사전에 인지하고 대비'할 수 있도록 외부 캘린더(구글, 애플 등) 연동을 지원하여, 별도의 자체 Push 서버 구축 없이 알림 환경을 구성한다.  
* **타겟 고객:** 영화 관람 빈도가 높고 혜택 정보를 적극적으로 탐색하는 체리피커 및 영화 마니아.  
* **플랫폼:** 모바일 반응형 웹 (Responsive Web), 앱 설치 불필요.

## **2\. 기획 배경 및 문제 정의**

* **정보의 파편화:** 3사 영화관 앱을 개별적으로 실행해 숨겨진 이벤트 페이지를 확인해야 하는 번거로움 존재.  
* **일정 관리의 부재:** 기존 커뮤니티나 앱은 단발성 알림에 그치며, 사용자가 다가오는 여러 영화의 예매 일정을 한눈에 파악하고 미리 본인의 일정에 등록해 두는 기능이 취약함.

## **3\. 핵심 컨셉 (Zero-Server Cost 전략)**

자체 Push 알림 서버(DB, 백엔드 로직, FCM 등)를 구축하지 않고, 클라이언트(유저)의 디바이스에 내장된 캘린더 앱을 활용하여 서버 유지 비용을 0원으로 수렴시키는 정적 아키텍처.

## **4\. MVP 주요 기능**

### **4.1. 3사 통합 타임라인 대시보드**

* 현재 진행 중이거나 예정된 3사 할인 이벤트 리스트를 시간순(Time-ordered)으로 정렬하여 제공.  
* 표시 정보: 극장명, 영화 제목, 이벤트 종류, 티켓 오픈 일시, 잔여 시간(D-Day/Time), 해당 극장 예매 링크.

### **4.2. 캘린더 사전 연동 (사전 대비 목적)**

사용자가 이벤트 발생 며칠 전부터 일정을 인지하고 대비할 수 있도록 두 가지 방식의 연동을 제공.

1. **단건 일정 추가 (Template URL / 권장):**  
   * 사용자가 특정 이벤트의 \[내 캘린더에 추가\] 버튼 클릭 시, Google Calendar 생성 URL(https://calendar.google.com/calendar/render?...)로 이동.  
   * 사용자의 캘린더에 즉시 등록되며, 티켓 오픈 시간 기준 10분 전 알림이 설정됨.  
   * **장점:** 게릴라성(1\~2일 전) 공지 이벤트도 사용자가 발견 즉시 딜레이 없이 캘린더에 반영 가능.  
2. **전체 일정 구독 (ICS URL 제공 / 보조):**  
   * 대시보드에서 제공하는 .ics URL을 사용자가 본인 캘린더에 '구독' 형식으로 추가.  
   * **단점:** 캘린더 앱 자체의 동기화 주기(12\~48시간)로 인해 단기 공지 이벤트 누락 발생 가능.

## **5\. 시스템 아키텍처 및 기술 스택**

* **Data Collection (크롤링 파이프라인):**  
  * **도구:** Python (BeautifulSoup, Selenium)  
  * **실행 환경:** GitHub Actions (스케줄러 기반 자동화, 무료)  
  * **로직:** 3사 이벤트 페이지 파싱 \-\> 이벤트 데이터를 정적 JSON 및 ics 파일로 변환 \-\> GitHub Repository에 배포.  
* **Front-end (정적 웹사이트):**  
  * **도구:** React, Vue 또는 Vanilla JS \+ CSS Framework (Tailwind 등)  
  * **호스팅:** GitHub Pages, Vercel, Netlify (무료 티어 활용)  
  * **로직:** 정적 JSON 데이터를 Fetch하여 사용자 브라우저에서 리스트 렌더링.

## **6\. 현실적 리스크 및 한계점**

1. **동적 페이지 크롤링의 취약성:**  
   * 3사 웹사이트는 DOM 구조가 수시로 변경되며, 특정 극장의 경우 봇 접근 차단(Anti-bot) 솔루션을 적용할 가능성이 높음. GitHub Actions의 IP 대역이 차단될 경우 프록시 우회 등의 추가 조치가 필요하며 이는 비용 발생을 초래할 수 있음.  
2. **선착순 이벤트 공지 시점의 불규칙성:**  
   * "며칠 전 미리 알림"이라는 기획 의도와 달리, 극장 측에서 티켓 오픈 당일이나 하루 전날 이벤트를 기습 오픈하는 경우가 있음.  
   * 이 경우 ICS 캘린더 구독 방식은 동기화 지연으로 무용지물이 되므로, 대시보드 자체의 주기적인 방문을 유도하거나 단건 캘린더 추가 기능을 메인으로 내세워야 함.
