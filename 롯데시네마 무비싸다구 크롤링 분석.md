# 롯데시네마 '무비싸다구' 크롤링 기술 분석 및 구현 전략

## 1. 이벤트 페이지 접근 특성

* **모바일 전용 환경**: 무비싸다구 이벤트 배너 및 상세 페이지는 모바일 웹 환경에서만 정상적으로 나타남.
* **리다이렉션 로직**: PC 브라우저의 User-Agent로 모바일 페이지에 접근하면 일반 이벤트 목록 페이지(https://www.lottecinema.co.kr/NLCHS/Event)로 강제 리다이렉션됨.
* **타겟 엔드포인트**: 모바일 메인 페이지인 https://www.lottecinema.co.kr/NLCMW/가 데이터 수집의 시작점임.

## 2. 기존 정적 파싱의 한계 (DOM 구조 분석)

* **숨겨진 동작 (Hidden Action)**: 배너 버튼의 HTML 태그가 `<a href="#">` 구조로 작성됨.
* **이벤트 위임**: 클릭 이벤트를 JavaScript로 처리하므로, 단순 HTML DOM 파싱 도구(BeautifulSoup 등)로 배너의 `<a>` 태그를 찾아도 목적지 URL을 추출하기 어려움.
* **가변적 파라미터**: 실제 이동하는 URL은 `https://www.lottecinema.co.kr/NLCMW/Event/EventTemplateSpeedMulti?eventId={가변ID}` 형태이며, 이 `{가변ID}`를 동적으로 알아내는 일이 핵심 과제임.

## 3. 데이터 원천(Data Source) 분석 및 신규 API 활용 전략

* **1단계 (세션 획득 및 MainEventID 추출)**:
  * 모바일 메인 페이지(https://www.lottecinema.co.kr/NLCMW/)에 GET 요청을 보내 반환된 HTML에서 메인 이벤트 ID(`MainEventID`, 예: 201210016922014)를 추출함.
  * 추출한 `MainEventID`를 이용해 상세 템플릿 페이지(`https://www.lottecinema.co.kr/NLCMW/Event/EventTemplateSpeedMulti?eventId={MainEventID}`)에 2차 GET 요청을 보냄. 이 과정은 웹 방화벽(WAF)을 통과하고 유효한 세션 쿠키를 발급받기 위한 필수 사전 작업임.
* **2단계 (API 요청 인코딩 특성 및 직접 호출)**:
  * 롯데시네마 이벤트 데이터 제공 API(`https://www.lottecinema.co.kr/LCWS/Event/EventData.aspx`)에 POST 요청을 전송함.
  * **인코딩 규칙**: 일반적인 JSON POST 전송이 아닌 **`multipart/form-data`** 형식을 요구함. 페이로드는 `paramList` 파라미터에 JSON 문자열 형태로 래핑하여 전달해야 함 (Python `requests` 라이브러리의 `files` 인자 활용).
  * **Payload 내부 구조 예시**: 
    ```json
    {
      "MethodName":"GetSpeedEventDetailMulti",
      "channelType":"MW",
      "osType":"A",
      "osVersion":"Mozilla/5.0 ... (모바일 User-Agent)",
      "EventID":"",
      "MainEventID":"201210016922014"
    }
    ```

## 4. 최종 크롤링 파이프라인 (API 직접 호출 기반)

위 분석을 바탕으로, 복잡한 HTML 파싱이나 브라우저 에뮬레이션을 최소화해 상세 데이터를 정확히 추출하는 과정은 다음과 같음.

1. **HTTP 요청 위장 (1차)**: 헤더의 User-Agent를 모바일 기기(iPhone/Android)로 명시해 `https://www.lottecinema.co.kr/NLCMW/`를 호출함.
2. **MainEventID 추출**: 정규표현식이나 문자열 파싱으로 웹페이지 내 인라인 주입된 데이터 블록을 탐색해 최신 `MainEventID`를 확보함.
3. **세션 쿠키 발급 (2차)**: 획득한 ID를 기반으로 상세 페이지(`EventTemplateSpeedMulti?eventId={MainEventID}`)에 접속해 정상적인 세션을 수립함.
4. **API POST 요청 (3차)**: 수립한 세션 쿠키와 multipart 형식을 적용해 `https://www.lottecinema.co.kr/LCWS/Event/EventData.aspx`로 API 데이터를 요청함.
5. **JSON 파싱 및 상세 데이터 추출**:
   * 반환된 JSON에서 `SpeedEventDetail` -> `ItemGroup` 배열, 그리고 그 하위의 각 `Items` 배열을 순회함.
   * `MovieNm` (영화 제목), `ProgressStartDate` (시작 날짜), `ProgressStartTime` (이벤트 오픈 시간), `Img5Url` (썸네일 이미지 주소), `MovieCd` (영화 코드) 핵심 데이터를 즉시 수집함. (기존 문서에 기술되었던 다운로드 수량이나 잔여 수량 정보는 실제 캘린더 생성 목적에 불필요하여 수집 항목에서 제외함)
6. **식별자 및 URL 조립**:
   * 고유 식별자는 `lotte-{MainEventID}-{MovieCd}` 형태로 생성함.
   * 이벤트 접근 링크인 `https://www.lottecinema.co.kr/NLCMW/Event/EventTemplateSpeedMulti?eventId={MainEventID}`와 이미지 URL을 결합해 최종 정규화 데이터 포맷을 완성함.
